Versioning

版本策略

遵循语义化版本规范，每次发布都有据可查

补丁版本 (x.x.1)

修复 bug、修正文案、改善兼容性。不改变核心安装逻辑，可直接覆盖安装。

小版本 (x.1.0)

新增功能：自检、升级提醒、日志导出、更多 Provider 模板等。向后兼容。

大版本 (2.0.0)

涉及安装架构变化、目录迁移、自动更新系统或跨平台支持。可能需要迁移步骤。

历史版本保留

旧版本不覆盖、不删除，保留下载入口。方便回滚与兼容性排查。

Release Standard

发布规范字段

每个版本发布时必须包含以下信息

字段 是否必须 说明
版本号 必须 语义化版本 MAJOR.MINOR.PATCH
发布日期 必须 ISO 8601 格式，方便用户判断新旧
平台 必须 Windows x64 / 后续扩展 macOS、Linux
下载文件名 必须 文件名必须包含版本号
SHA256 必须 安全校验基础
更新摘要 必须 告诉用户这版改了什么
升级说明 建议 是否可直接覆盖安装、是否需要迁移
已知问题 建议 透明说明当前版本的已知限制
